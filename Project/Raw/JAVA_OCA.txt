--Diamond Problem:-
----Derived classes can get multiple implementations for the same method signature and hence multiple inheritance not allowed in java.
---- Class A {}, Class B extends A {}, Class C extends A{}, class D Extends B,C{} -- Here D doesnt decide which class method it has to execute. causing ambuguity.

----But from Java8 with default methods introduction in interfaces diamond problem resurfaced.
----To fix this, Java requires the implementing class to explicitly override the conflicting method and decide which one to use: like Interface.super.method();

--Upcasting:
----When you upcast, you treat a subclass object as its superclass.
-----You can only access the members defined in the superclass (even though the object is actually a subclass).

--Downcasting (Superclass → Subclass)
----When you downcast, you tell Java to treat a superclass reference as a subclass reference.
---- Safe only if the object is actually an instance of that subclass.
---- Allows access to subclass-specific properties.

--Key Difference: Properties vs Methods
----Properties (fields): Access depends on the reference type, not the actual object.
----Methods: Overridden methods are resolved at runtime (dynamic dispatch), so the actual object’s method runs.

--Casting changes what you can access through a reference.

----Downcasting lets you access subclass-specific members, but only if the object truly is that subclass.

Java Memory management:

--JVM divides memomry into several runtime areas. out of which main are

--Heap: stores objects and arrays. shared across all the threads.
--Stack: stores method calls and local variables. each thread has it's own stack.
--Metaspace: contains class metadata. earlier it was permgen
--PC: each method instruction stored here
--native method stack: non java method calls stored here.

How memory is managed?
-- Object creation: reference is created in stack and object is created in heap.
-- Garbage colection: JVM autolatically reclaims memory from teh objects which are no longer refrenced.
-- Generational Heap structure of GC contains three sections:
    -- young generation: newly created objects
	--old generation: long-lived objects
	--Metaspace: conatins class metadata

--Metaspace contains metadata for classes. replacement of permgenspace from java8
-- Meta space is outside of heap where as permgenspace is inside heap.
-- Meta space size is dynalic and grows as needed where as permgenspace is fixed and needed amnual tuning.
-- with Meta Space outof memory error is rare only happens if system memory is ehausted.

Java Memory Model: this is how JVM handles memory in caes multi threading environment.
- Defines how threads interact through memory.
- Specifies when changes made by one thread become visible to others.
- Provides rules for synchronization and atomicity.
- Prevents concurrency bugs like race conditions and stale data.

-Happens-Before Relationship:- Defines ordering rules for operations across threads.
