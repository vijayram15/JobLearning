--instanceOf:
---- a instanceOf b means a subclass of b or a implements b.
---- compilation check happens only when instanceOf is called on Class, for interface it will be checked at runTime.
---- class resolution happens at compile time where as interface resolution happens at runtime.

--synchronizedMap vs ConcurrentHashMap
----synchronizedMap is a normal map wrapped with synchronized methods, where ConcurrentHashMap is conurrenct version implementaion of HashMap.
----synchronizedMap is fail fast throws concurrentModificationException on modifying the Map, where as ConcurrentHashMap is fail safe doesnot throw concurrent Modification exception.
----synchronizedMap allows null key and many null values, where as ConcurrentHashMap does not allow null.
----synchronizedMap locks entire Map on read/write, where as ConcurrentHashMap allows bucket/segment/bin level locking using CAS operations.
----synchronizedMap performs better at low concurrent environments whereas ConcurrentHashMap performs in higher concurrent environments.

--virtual method:
----it's the method overidden in subclass but invoked at runtime based on object type.
---- non-final, non-static, non-private methods are virtual methods only.

--equals contract
----reflexive:- x.equals(x) true
----symmetric:-x.equals(y)==y.equals(x) true
----transitive: x.equals(y), y.equals(z), then x.equals(z) true
----x.equals(null) always returns false

--hashcode() contract
----if two objects are equal with equals opertaion then thier hashcodes should be equal.
----if two objects are not equal then their hashcodes may or may not be equal.

--Nested Classes:
----Member Inner class--class defined at instance member level.
----Local Inner Class--class defined inside method
----anonymous inner class---local inner class without name
----static inner class--class defined at static variable level.

----Access Modifiers are allowed for member inner class and static inner class only because other two are local already.
----except anonymous inner classes remaing can extend any number of classes/interfaces. Anonymous inner class should have exactly one superclass/interface.
----except anonymous inner class remaining can be abstract and final.
----except static classes remaining can access instances of outerclass and can't declare static methods.
----local inner and anonymous inner classes access local variables if and only if they are final or effectively final.

streams vs parallel streams:-
--streams run on single thread, where as parallel streams splits the tasks on multiple threads using forkjoin pool.
--streams best for small datasets/lightweight operations, parallel streams suited for larger datasets/CPU-senstive operations.
--streams preserves encountered order, but parallel streams wont preserve the order unless forced explicitely.

streams vs collections:
--collections are in-memory datastructures store and manage elements, streams are pipeline for processing the data from collections.
--elements must be computed and stored before. Lazy computation.
--even after processing collections will be available, streams once used wont be available further.
